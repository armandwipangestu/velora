## What is Velora?

It is an open-source blog template. The core tech stack uses Next.js, Tailwind CSS, Velite, MDX, <a className='text-[#10b981]' href="https://shiki.style/" target="_blank">Shiki</a>, and <a className='text-[#10b981]' href="https://rehype-pretty.pages.dev/" target="_blank">Rehype Pretty Code</a>. With Velite, you can create a blog with a type-safe data layer, as it is defined using TypeScript and Zod for schema validation. See the Velite <a className='text-[#10b981]' href="https://velite.js.org/guide/introduction" target="_blank">docs</a> for more information.

Velora provides a blog template with all the essential features built-in. This means you don't need to build a content pipeline from scratch; Velora includes pre-built components and utilities to help you get your blog running immediately.

However, the previous setup required significant manual configuration for frontmatter and syntax highlighting (using highlight.js), and the code blocks were limited to basic highlighting. After researching ways to create a richer code block experienceâ€”including features like line numbers, word/line highlighting, and diff notationâ€”the author discovered Shiki and Rehype Pretty Code. These libraries are the core of modern code block tools like <a className='text-[#10b981]' href="https://expressive-code.com/" target="_blank">Expressive Code</a> and <a className='text-[#10b981]' href="https://nextra.site/" target="_blank">Nextra</a>.

Following that tutorial made everything click. The most impressive part of this setup is how easily the plugins can be configured and how Velite acts as a seamless content pipeline, making MDX content easy to maintain, parse, and extend with custom React components.

## Why Choose Shiki and Rehype Pretty Code?

After doing some research, the author explored several options for syntax highlighting, such as <a className='text-[#10b981]' href="https://highlightjs.org" target="_blank">highlight.js</a>, <a className='text-[#10b981]' href="https://prismjs.com/" target="_blank">Prism.js</a>, and <a className='text-[#10b981]' href="https://codehike.org/" target="_blank">Code Hike</a>. However, after reviewing their configurations, the author found that Shiki and Rehype Pretty Code were the best options.

What makes Shiki particularly interesting is that it uses the same engine that powers VS Code: <a className='text-[#10b981]' href="https://code.visualstudio.com/api/language-extensions/syntax-highlight-guide#textmate-grammars" target="_blank">TextMate Grammars</a>. This provides some of the most accurate and beautiful syntax highlighting available; you can read more about Shiki's history <a className='text-[#10b981]' href="https://nuxt.com/blog/shiki-v1" target="_blank">here</a>. The best part is that Shiki is static rather than runtime-based. Itâ€™s fast and doesn't require loading additional files because itâ€™s bundled during the build processâ€”unlike highlight.js or Prism.js, which typically run in the browser and can impact performance.

Rehype Pretty Code is a plugin that transforms Markdown into HTML with specialized classes and attributes. It provides features like line numbers, line highlighting, and word highlighting. The ecosystem also supports advanced features like diff notation via <a className='text-[#10b981]' href="https://shiki.style/packages/transformers" target="_blank">@shiki/transformers</a> and interactive TypeScript code samples via <a className='text-[#10b981]' href="https://shiki.style/packages/twoslash" target="_blank">@shiki/twoslash</a>.

Finally, the author observed that many modern documentation sites are migrating to Shiki and Rehype for their superior syntax highlighting capabilities.
